## Category Strategy

LinkedIn categories are broad routing labels, not my actual offer. I should keep categories that
can plausibly route backend, cloud, data, information, consulting, and custom-system work. I should
remove categories that mostly attract isolated websites, student assignments, standalone games, or
generic mobile/ecommerce requests.

Recommended keep:

- Cloud Application Development
- Custom Software Development
- Information Management
- IT Consulting

Recommended remove:

- Web Development
- Application Development

Recommended add if LinkedIn allows it:

- Cloud Management
- Database Development
- Data Reporting
- Business Analytics
- Software Testing
- Technical Support
- Cybersecurity

Do not add:

- Mobile Application Development
- broad web-only categories
- categories that mainly route design, content, marketing, simple site refreshes, or student work

## Request Triage Rule

Good fit:

- backend-owned AI workflow, RAG, transcript/call/document analysis, approval flow, CRM action, or
  internal operator workflow;
- CRM/ERP/1C/banking/accounting/API/database integration with contracts, retries, logs, and
  rollout notes;
- FastAPI/PostgreSQL backend, admin workflow, audit trail, worker/outbox, tests, Docker, CI, and
  runbook;
- DevOps/recovery/hardening work with deploy, logs, health checks, smoke tests, backup/restore, or
  rollback.

Not a fit unless there is a real backend/integration system to own:

- isolated website refresh;
- student or course assignment;
- standalone game clone;
- generic mobile/ecommerce app;
- vague request with no system boundary and no verifiable success condition.

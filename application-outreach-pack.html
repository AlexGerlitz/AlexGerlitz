Vacancy Triage

Role laneApply now: Python/backend, QA/API Python, integration/API/CRM, backend-owned AI workflow. Adapt first: backend role with unclear first task or broad AI wording. Skip: senior platform, Kubernetes/Terraform-first, onsite-only, pure content.
First resultApply now: one workflow slice, API check, adapter, admin path, or verification route. Adapt first: broad system where the first slice can be named. Skip: success cannot be tested or handed off.
Stack overlapApply now: Python, FastAPI, PostgreSQL/SQL, REST/OpenAPI, pytest, Docker Compose, GitHub Actions. Adapt first: adjacent stack with enough API/data workflow ownership. Skip: no Python/API/data/backend ownership.
Work sampleApply now: one public work-sample link directly matches the vacancy. Adapt first: the review path needs a short framing note. Skip: evidence would require private data or fake claims.

First Task Shape

Ask for one bounded workflow slice with input, state, output, verification, and handoff. Good examples: one API check, one admin queue path, one adapter contract, one RAG evaluation path, one retry/audit path, or one smoke-checked backend route.

Communication Loop

  1. Pick one lane: Python backend/internal tools, QA/API verification, integration/API/CRM, or backend-owned AI workflow.
  2. Lead with one work sample, not a pile of links.
  3. State one first useful result with input, state, output, verification, and handoff.
  4. Keep Docker/CI and AI as supporting evidence unless the role clearly asks for them.
  5. Ask for the first task shape, review owner, success check, timeline, and compensation band if shareable.
